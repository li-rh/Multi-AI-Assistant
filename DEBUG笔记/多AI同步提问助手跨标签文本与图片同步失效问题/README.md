## 资产链路调试 6 行速查表
1. 同一次操作出现两次“写入共享资产消息” → **源端重复广播**（优先检查 `paste + change` 双触发）。
2. 先有“命中内容去重，跳过广播”且无第二次写入 → **去重生效**，非源端重复。
3. 同一 `assetMessageId` 在目标页出现两次“开始远端资产注入” → **目标端幂等失效**。
4. 目标页仅一次“开始远端资产注入”，但聊天框出现两张图 → **站点前端自身二次落图**（非脚本二次分发）。
5. 出现“尝试 paste 注入结果: true”后仍紧跟“改走 file input 注入” → **注入分支异常并行**（需加站点特判）。
6. 远端注入后很快又看到“监听到 file input change 事件”并触发广播 → **抑制窗口不足或站点延迟事件**。

## v3.3-dev 最小回归脚本（通义 + 元宝）

### 前置条件
1. Tampermonkey 菜单开启“资产链路日志”。
2. 只保留 2 个标签页：`qianwen.com` 与 `yuanbao.tencent.com`。
3. 在发送端只执行**单次**操作，等待 3~5 秒再做下一次。

### 用例 A：图片（发送端通义，目标元宝）
1. 在通义输入区粘贴一张图片。
2. 发送端预期日志：
	- 一次“捕获本地资产事件”
	- 一次“通过内容去重检查，准备广播”
	- 一次“写入共享资产消息”
3. 目标端预期日志（元宝）：
	- “开始远端资产注入”
	- “paste 注入后校验结果”
	- 若校验失败才出现“file input 注入后校验结果”
4. 判定通过：目标端聊天框仅 1 张新图。

### 用例 B：非图片文件（发送端通义，目标元宝）
1. 在通义上传一个 `txt/pdf/docx` 任意文件。
2. 目标端预期日志（元宝）：
	- “开始远端资产注入”
	- 不应出现 paste 成功即结束（非图片默认走 file input）
	- 出现“file input 注入后校验结果”
3. 判定通过：目标端出现对应文件附件（或文件名芯片）。

### 用例 C：非图片文件（发送端元宝，目标通义）
1. 在元宝上传一个 `txt/pdf/docx` 任意文件。
2. 目标端预期日志（通义）：
	- “开始远端资产注入”
	- 允许出现“未立即找到可用 file input，尝试触发上传入口”
	- 之后应出现“file input 注入后校验结果”
3. 判定通过：通义端出现文件附件，且无重复附件。

### 失败快速归因（按日志关键词）
1. “当前资产类型按站点规则禁用 file input，结束注入流程”
	- 说明规则把该类型关掉了（应仅用于图片防重，不应影响非图片）。
2. “未找到可用 file input，无法执行回退注入”
	- 说明站点上传入口/selector 仍未覆盖，需要补站点上传按钮或容器定位。
3. 连续两次“写入共享资产消息”且 `fingerprint` 不同
	- 说明源端捕获到两份不同内容（可能站点二次转码），需额外站点侧去重策略。
