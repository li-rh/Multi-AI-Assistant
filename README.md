# 多AI同步提问助手

> 一个面向多模型对比场景的 Tampermonkey 脚本：在任意已支持 AI 页面发起一次提问，即可自动同步到其他已勾选页面，支持文本、图片与通用文件。

## 为什么做这个脚本（开发动机）

在日常对比 ChatGPT / Claude / Gemini / 豆包 / 千问等模型时，最耗时的不是思考，而是重复劳动：

- 同一问题反复复制粘贴
- 多个标签页来回切换
- 图片/文件在每个站点重复上传

本项目的目标很明确：**把重复操作压缩到一次动作**，把时间留给“比较答案质量与推理差异”。

---

## 亮点速览

### 1) 文本同步更稳：主链路 + 兜底链路

- 主链路：拦截 XHR / fetch 请求，按站点协议提取真实提问内容
- 兜底链路：监听本地发送行为（Enter / 按钮点击）
- 结果：即使某站点网络层特征变化，也有较高概率继续可用

### 2) 资产同步可控：图片/文件跨标签广播

- 支持入口：`paste`（粘贴）与 `drop`（拖拽）
- 接收端按“站点注入策略”自动选择注入方式（可按站点独立配置）
- 注入后会做 DOM/网络联合校验，降低“看起来注入了但实际没上传”的误判

### 3) 去重与防回环：减少重复发送

- 发送端：基于内容指纹的短窗口去重
- 接收端：基于 `assetMessageId` 的幂等消费
- 远端注入后本地短抑制窗口，避免“回填事件再次广播”

### 4) 多标签协同：状态同步 + 活跃标签心跳

- 可选择“跨标签共享选择状态”或“每页独立选择”
- 脚本维护活跃标签心跳，自动清理过期标签状态

### 5) 可观测与可诊断

- 可切换普通调试日志 / 资产链路日志
- 内置启动自检（可选严格模式），异常时更容易定位

---

## 核心功能

### 文本同步

- 自动识别当前站点并提取提问内容
- 将消息广播到已选目标标签页
- 目标页自动注入输入框并触发发送

### 图片/文件同步

- 监听本页粘贴和拖拽文件事件
- 将文件转为可跨标签传输的数据结构后广播
- 目标页自动注入上传动作（`drop` 或 `paste`）

### 可视化控制面板

- 右下角悬浮球入口
- 模型勾选、全选/取消全选
- 常用模型显示管理
- 当前选中目标数徽标提示

### 设置项

- 动画样式切换（Spin / Aurora）
- 选择状态同步开关
- 站点级资产注入策略（drop/paste）
- 一键应用推荐策略
- 调试日志 / 资产链路日志 / 严格模式开关

## 快速开始（1 分钟）

1. 安装 Tampermonkey。
2. 安装并启用 `多AI同步提问助手.js`。
3. 打开 2 个及以上已支持的 AI 站点页面。
4. 点击右下角悬浮球，勾选目标模型。
5. 在任一页面发送文本，或粘贴/拖拽图片/文件。
6. 观察其他已选页面自动同步。

---

## 推荐使用方式

- 先打开常用页面再勾选，首次协同更顺畅。
- 大文件同步耗时更长，属于正常现象。
- 若某站点上传策略不稳定，可在设置里将该站点改为另一注入方式（drop ↔ paste）。
- 当你只想单页工作，取消其他模型勾选即可。

## 稳定性设计（给进阶用户）

- **消息时效控制**：过期消息自动丢弃，避免旧任务误触发。
- **幂等消费**：同一资产消息只处理一次，降低并发重复。
- **状态隔离**：文本与资产分离通道，降低链路互相干扰。
- **启动自检**：关键配置/工具函数缺失时快速暴露问题。

## 常见问题（FAQ）

### Q1：为什么有时某站点没自动发出？

- 可能是该站点输入框结构临时变化或页面尚未完全就绪。
- 处理建议：刷新目标页重试；确认该模型已勾选且页面在线。

### Q2：为什么文件同步偶尔不生效？

- 部分站点对“非用户手势上传”限制更严格。
- 处理建议：切换该站点注入策略（drop/paste）；开启资产链路日志定位。

### Q3：为什么会出现重复触发？

- 脚本已内置内容指纹去重与回环抑制；若仍出现，可开启日志并反馈复现步骤。

## 自测清单

1. 文本同步是否成功到全部已选页面
2. 粘贴图片是否可跨页上传
3. 拖拽文件是否可跨页上传
4. 不同站点注入策略（paste/drop）切换后是否生效
5. 选择状态同步开关是否符合预期
6. 日志开关与严格模式是否正常工作
